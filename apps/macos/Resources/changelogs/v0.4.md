## What's new in Myna 0.4

### Voice engine reliability (0.4.3)
Fixed a setup issue where the on-device voice engine could install an
incompatible version of its text-processing library and then fail every read
with a 502 — the cause of "the voice works on one Mac but not another." Setup
now installs a known-good, pinned version, and repairs an already-broken engine
on the next run. Added diagnostics to the read pipeline to pinpoint any
remaining "stops responding after a few reads" reports.

### Faster install & upgrade (0.4.2)
Setting up the voice engine is dramatically quicker:
- Re-installs and upgrades now **reuse** the engine that's already on disk
  instead of re-downloading ~600 MB of Python packages every time — seconds
  instead of minutes.
- First-time setup installs with **uv** (parallel downloads + a much faster
  resolver, ~3× quicker than before) and pulls the voice model over
  HuggingFace's high-performance Xet transfer.

### No more crash after the first read (0.4.1)
Fixed a macOS 26 crash that quit Myna the instant the first chunk of audio
finished playing. Reading is stable from start to finish again.

### One service for the whole voice stack (0.4.0)
The daemon now starts and supervises the on-device voice engine itself, so
there's a single thing to manage. `brew services restart myna-daemon` (or the
menu-bar **Restart Daemon**) restarts everything, and stopping the daemon stops
the engine. No more separate engine background service to think about.

Existing installs migrate automatically on first launch — the old engine
LaunchAgent is retired and the daemon takes over.

### Reliability, end to end (0.3.3 – 0.4.0)
- The daemon never gets stuck after an interrupted read — audio keeps working.
- Setup downloads the voice model fully before it finishes, so the first read
  just works.
- The Setup and Onboarding windows no longer crash when you close them.
- Fixes for recording a hotkey, the Settings window coming to the front,
  popover scrolling, and the "What's New" link.
